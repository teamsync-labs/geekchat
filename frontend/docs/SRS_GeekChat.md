# Что мы строим

GeekChat — открытый учебный проект по разработке веб-приложения.

Наша цель — не сделать очередной аналог Zoom.

Наша цель — научиться создавать продукты в команде, используя реальные инженерные практики.

После завершения проекта каждый участник сможет показать работодателю:

- публичный GitHub-репозиторий;
- свои Pull Request;
- историю коммитов;
- участие в Code Review;
- работающий продукт;
- страницу команды;
- сертификат участия;
- рекомендательное письмо.

# Что считается успешным MVP

Через 3–6 недель должно работать следующее.

Пользователь А

↓


получает ссылку

↓

отправляет ссылку пользователю Б

↓

пользователь Б открывает ссылку

↓

начинается видеозвонок.

Если этот сценарий работает стабильно —

MVP считается завершенным.

---

# Наши принципы

Мы пришли сюда не просто написать код.

Мы учимся работать как инженерная команда.

Поэтому придерживаемся следующих правил.

- Все изменения проходят через систему контроля версий.
- Работа ведётся небольшими Pull Request.
- Каждый Pull Request проходит Code Review.
- Архитектурные решения обсуждаются открыто.
- Ошибаться нормально.
- Не задавать вопросы — хуже, чем задать "глупый" вопрос.
- Документация развивается вместе с проектом.
- Любой участник может предложить улучшение продукта или процесса.

# Архитектура MVP

```
Browser
      │
      │ HTTPS / WebSocket
      ▼
 FastAPI
      │
      ▼
 PostgreSQL

Browser ←──── WebRTC P2P ────→ Browser
```

FastAPI отвечает только за:

- авторизацию;
- создание комнат;
- signaling.

Видео и аудио идут напрямую между браузерами.

# Definition of Done

MVP считается завершенным, если:

- работает сценарий "создать ссылку → отправить → начать видеозвонок";
- проект доступен по публичному адресу;
- код находится в открытом GitHub;
- все основные изменения прошли Code Review;
- документация актуальна;
- каждый участник может показать свой вклад работодателю.
