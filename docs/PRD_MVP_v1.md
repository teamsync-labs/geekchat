## 1. О проекте

GeekChat — учебный командный проект стажировки TeamSync Labs. Цель — не сделать очередной аналог Zoom, а научиться создавать продукты в команде, используя реальные инженерные практики.

После завершения каждый участник получает: публичный GitHub-репозиторий с историей своей работы, участие в Code Review, работающий продукт, сертификат и рекомендательное письмо.

---

## 2. Цель MVP

> **Главный сценарий:** Пользователь А создаёт ссылку → отправляет пользователю Б → Б открывает ссылку → начинается видеозвонок.

Если этот сценарий работает стабильно — MVP считается завершённым.

## 4. Что входит в первый релиз (Scope MVP)

### 4.1 Бэкенд

- Регистрация и авторизация пользователей (JWT)
- Создание комнаты → уникальная ссылка (UUID)
- `GET /api/rooms/{room_id}` — проверка существования комнаты
- WebSocket signaling — обмен SDP offer/answer и ICE candidates
- Хранение данных в PostgreSQL
- Миграции через Alembic

### 4.2 Фронтенд

- Лендинг — публичная страница с описанием продукта
- Форма регистрации (поля: Имя, Email, Пароль, Подтвердить пароль)
- Форма входа
- Страница создания комнаты и копирования ссылки
- Страница звонка: локальное и удалённое видео, управление камерой и микрофоном
- P2P видео через PeerJS + WebRTC
- Стили через Tailwind CSS

### 4.3 DevOps / Инфраструктура

- Docker Compose для локального запуска
- GitHub Actions: линтинг + тесты на каждый PR
- Автодеплой при мерже в main
- Публичный HTTPS-адрес

### 4.4 QA

- Smoke-тесты основного пользовательского сценария
- Тестирование форм регистрации и входа (позитивные и негативные сценарии)
- Баг-репорты по утверждённому шаблону
- Регрессионное тестирование после фиксов

---

## 5. Что не входит в первый релиз

> Всё перечисленное ниже — это сознательное решение, а не упущение. Эти функции могут войти в следующий релиз.

- Групповые звонки (более 2 участников) — только P2P
- Текстовый чат во время звонка
- Запись звонков
- Демонстрация экрана (screen share)
- TURN-сервер (звонок может не работать за симметричным NAT)
- Личный кабинет с историей звонков
- Мобильное приложение
- Сброс пароля по email
- Refresh-токены (автоматическое обновление сессии)
- Уведомления
- Административная панель

## 6. Основной пользовательский сценарий

Для MVP выбран сценарий 3 — оба пользователя зарегистрированы:

| Шаг | Действие | Ожидаемый результат |
|---|---|---|
| 1 | Пользователь А регистрируется | Аккаунт создан, выдан JWT-токен |
| 2 | Пользователь А создаёт комнату | Получает ссылку вида `/room/abc123` |
| 3 | А отправляет ссылку пользователю Б | Ссылка скопирована в буфер |
| 4 | Пользователь Б регистрируется и открывает ссылку | Браузер запрашивает доступ к камере и микрофону |
| 5 | Оба дают доступ к камере и микрофону | Устанавливается P2P соединение |
| 6 | Видеозвонок активен | Оба видят и слышат друг друга |

## 8. Definition of Done — критерии готовности MVP

MVP закрыт, когда выполнены **все** пункты:

- [ ] Сценарий "создать ссылку → отправить → начать видеозвонок" работает стабильно
- [ ] Проект доступен по публичному HTTPS-адресу
- [ ] Код находится в открытом GitHub-репозитории
- [ ] Все изменения прошли Code Review (минимум 1 ревьюер)
- [ ] Нет блокирующих и критичных багов
- [ ] Документация (README, docs/) актуальна
- [ ] Каждый участник может показать свой вклад работодателю
- [ ] Продакт подписал приёмку

---

## 9. Что планируем после MVP

После успешной сдачи MVP возможны следующие направления развития:

- Переход на React (SPA-архитектура)
- Личный кабинет с историей звонков
- Текстовый чат во время звонка
- SFU на Go — групповые звонки
- TURN-сервер для устойчивости в сложных сетях
- Административная панель

Состав следующего релиза определяется совместно после закрытия MVP.
