# 01. Постановка задач

Каждая задача на доске kanban описывается **в указанных разделах**.

Порядок обязателен — так проще читать, ревьюить и оценивать scope.

---

## Структура описания

### 1. User story

Кто, что хочет и зачем. Обычно 1–2 истории, не больше.

```markdown
**Как** <роль>,
**я хочу** <действие>,
**чтобы** <ценность>.
```

### 2. Суть

2–4 предложения: что делаем технически и как это связано с продуктом. Без деталей реализации.

### 3. Зависимости

Что должно быть готово до начала или параллельно. Указать блокеры и что эта задача разблокирует.

### 4. Делаем

Конкретный scope: модели, API, экраны, файлы — списком.

### 5. Не делаем

Явные границы: что сознательно не входит в задачу (чтобы не раздувать PR).

### 6. Критерии приёмки

Чеклист «готово, когда…». Проверяемые пункты, по которым можно закрыть задачу.

---

## Пример (GEE-42: Room)

### User story

**Как** автор встречи,  
**я хочу** создать комнату и получить ссылку,  
**чтобы** отправить её гостю и начать видеозвонок.

**Как** гость,  
**я хочу** открыть ссылку и убедиться, что встреча существует,  
**чтобы** перейти к подключению к звонку.

### Суть

Room — запись в PostgreSQL о встрече с уникальным `room_id`. По нему строится ссылка; гость проверяет, что встреча существует. Видео в БД не хранится — только метаданные комнаты.

### Зависимости

- рабочий `db/database.py`, PostgreSQL, `get_db`
- роутеры подключены в `main.py`
- auth для `POST /api/rooms` *(временно допустим stub — с пометкой в PR)*

**Блокирует:** WebSocket signaling (нужен `room_id`)

### Делаем

- модель `Room` (`id`, `created_at`, `created_by`, `status`, `expires_at`)
- schema создания/ответа
- service: create, get, проверка joinable
- `POST /api/rooms` — создать комнату
- `GET /api/rooms/{room_id}` — проверить существование
- ответ API: `room_id`, `join_url`, `status`, `created_at`

### Не делаем

- WebSocket / signaling (SDP/ICE)
- чат, запись, screen share
- сохранение имени гостя в БД

### Критерии приёмки

- [ ] `POST /api/rooms` возвращает `room_id` и `join_url`
- [ ] `GET /api/rooms/{room_id}` → 200 для активной комнаты
- [ ] `GET /api/rooms/{unknown_id}` → 404
- [ ] endpoints видны в Swagger
- [ ] минимум 1 тест: create → get by id
